# User Profile System Design

> This document describes the provider-neutral profile design. The active AI
> runtime uses the provider selected through `AIProviderCatalog`; it does not
> expose a separate Claude-specific SDK path.

## Overview

The User Profile System enables different types of users (developers, analysts, architects) to have customized annotation experiences tailored to their specific needs and workflows.

## Profile Types

### 1. Developer Profile

**Purpose**: Technical implementation and code quality

**Key Features**:

- **Annotation Focus**: Bugs, fixes, improvements, performance
- **Default Tags**: `bug`, `fix`, `improvement`, `performance`, `refactor`
- **Provider-neutral AI prompts**:
    - "Analyze this bug and suggest a fix"
    - "Review performance implications"
    - "Suggest refactoring opportunities"
- **Templates**:
    - Bug Report Template
    - Performance Issue Template
    - Code Improvement Template
    - Technical Debt Template

**Settings**:

```json
{
    "profile": "developer",
    "defaultSeverity": "warning",
    "prioritizeTags": ["bug", "fix", "performance"],
    "aiContext": "You are assisting a developer with code quality and bug fixes",
    "autoSuggestFixes": true,
    "showCodeMetrics": true
}
```

### 2. UserProfileManager Class

```typescript
export class UserProfileManager {
    private context: vscode.ExtensionContext;
    private profiles: Map<string, UserProfile>;
    private activeProfile: UserProfile | null = null;

    constructor(context: vscode.ExtensionContext) {
        this.context = context;
        this.profiles = new Map();
        this.loadProfiles();
    }

    public getActiveProfile(): UserProfile | null {
        return this.activeProfile;
    }

    public setActiveProfile(profileId: string): void {
        const profile = this.profiles.get(profileId);
        if (profile) {
            this.activeProfile = profile;
            this.applyProfileSettings(profile);
            this.context.globalState.update('activeProfile', profileId);
        }
    }

    public getProfileTemplates(): string[] {
        return this.activeProfile?.settings.templateIds || [];
    }

    public getAIContext(): string {
        return this.activeProfile?.settings.aiContext || '';
    }

    private applyProfileSettings(profile: UserProfile): void {
        // Update workspace configuration with profile settings
        const config = vscode.workspace.getConfiguration('annotation');
        config.update('defaultSeverity', profile.settings.defaultSeverity);
        config.update('disabledTags', this.getDisabledTags(profile));

        // Notify other components of profile change
        vscode.commands.executeCommand('annotation.profileChanged', profile);
    }

    private getDisabledTags(profile: UserProfile): string[] {
        // Return all tags except those prioritized by the profile
        const allTags = [
            'bug',
            'fix',
            'improvement',
            'todo',
            'question',
            'requirement',
            'feature',
            'documentation',
            'architecture',
            'security',
            'scalability',
        ];
        return allTags.filter((tag) => !profile.settings.prioritizeTags.includes(tag));
    }
}
```

### 3. Provider-neutral AI integration

#### Profile-Aware Prompts

When creating annotations with AI suggestions, the system will:

1. **Prepend Profile Context**: Add the profile's AI context to prompts
2. **Filter Suggestions**: Based on profile's preferred tags and focus areas
3. **Template Selection**: Offer profile-specific templates first

Example integration:

```typescript
// In annotation creation with AI
async function getAISuggestion(code: string, profile: UserProfile): Promise<string> {
    const context = profile.settings.aiContext;
    const prompt = `${context}\n\nAnalyze this code and suggest relevant annotations:\n${code}`;

    // Send through the provider selected by the active AI catalogue.
    return await activeProvider.suggest(prompt);
}
```

## User Experience Flow

### 1. First-Time Setup

1. Extension detects no active profile
2. Shows welcome message with profile selector
3. User selects profile type (Developer/Analyst/Architect)
4. Profile settings are applied automatically

### 2. Profile Switching

1. Click profile indicator in status bar
2. Select new profile from quick pick
3. Workspace updates with new profile settings
4. Annotation views refresh with profile filters

### 3. Creating Annotations

1. User triggers annotation creation
2. System offers profile-specific templates
3. AI suggestions use profile context
4. Annotation is tagged with profile defaults

### 4. Viewing Annotations

1. Tree view shows profile-prioritized tags
2. Filters default to profile preferences
3. Kanban columns match profile workflow

## Migration Strategy

1. **Backwards Compatibility**: Existing annotations work without profiles
2. **Default Profile**: Users without profiles get "developer" as default
3. **Profile Detection**: Suggest profile based on existing annotation patterns
4. **Gradual Adoption**: Profiles are optional, full functionality without them

## Future Enhancements

1. **Team Profiles**: Share profiles across team members
2. **Profile Analytics**: Track annotation patterns per profile
3. **AI Learning**: Provider workflows can learn from profile-specific annotations
4. **Custom Profiles**: Users can create entirely custom profiles
5. **Profile Templates**: Export/import profile configurations
