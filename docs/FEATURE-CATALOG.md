# Feature catalogue and SDK audit

Audited against the source tree on 2026-07-13. The manifest version at the
time of the audit is `1.4.3`. This catalogue describes the repository state,
not the version currently available from a marketplace.

The audit uses the following evidence, in descending order of confidence:

1. runtime wiring in [`src/extension.ts`](../src/extension.ts);
2. providers, managers, views and the transactional store under [`src/`](../src/);
3. automated tests under [`src/test`](../src/test),
   [`src/transactional/__tests__`](../src/transactional/__tests__) and
   [`src/anchoring/__tests__`](../src/anchoring/__tests__);
4. contributions in [`package.json`](../package.json);
5. release history in [`CHANGELOG.md`](../CHANGELOG.md).

Documentation or a manifest contribution alone is not treated as proof that a
workflow is operational.

## Status legend

| Status      | Meaning                                                                                                            |
| ----------- | ------------------------------------------------------------------------------------------------------------------ |
| **Stable**  | Wired into the active extension and covered by an automated test or a directly verifiable implementation contract. |
| **Partial** | Usable code exists, but wiring, parity, safety, accessibility or automated validation is incomplete.               |
| **Planned** | No complete runtime capability exists; the work is recorded in [`tasks.jsonp`](../tasks.jsonp).                    |
| **Blocked** | The requested interaction cannot be implemented with the current stable Microsoft VS Code API.                     |

“Validation” below names existing tests; it does not assert that an unexecuted
working tree is green. “Introduced” is taken from the changelog when it is
knowable. Task links open the ledger; search for the displayed task ID.
