# Release Checklists

Quick-reference checklists for each release phase.
Full procedure: [RELEASING.md](./RELEASING.md).

---

## Pre-Commit Checklist

- [ ] `npm run lint:ci` exits 0 (zero warnings, zero errors).
- [ ] `npm run typecheck` exits 0 (zero type errors).
- [ ] `npm run compile` exits 0 and `dist/extension.js` is up to date.
- [ ] `git diff --stat` contains no `.env`, `logs/`, `dist/`, or `out/` files.
- [ ] No hardcoded secrets, API keys, or tokens in staged files.
- [ ] No em-dash characters (U+2014) in any committed file.
- [ ] No AI authorship markers (`Generated by`, `Co-Authored-By: Claude`, etc.).
- [ ] `package.json` version matches the intended release version.
- [ ] `CHANGELOG.md` has a section for the current version with today's date.
- [ ] `media/` contains no orphan or unreferenced binary files.

---

## Pre-Push Checklist

- [ ] `git status` shows a clean working tree (or only intentionally staged changes).
- [ ] Commit message follows Conventional Commits: `type: short description`.
- [ ] `git log --oneline -3` confirms the intended commit is HEAD.
- [ ] `git remote -v` shows the correct `origin` URL.
- [ ] No sensitive files appear in `git show --stat HEAD`.
- [ ] Branch is `main` (or a named feature/hotfix branch for PRs).

---

## Pre-Release Checklist

- [ ] `package.json` version bumped to the new semver value.
- [ ] `CHANGELOG.md` `[Unreleased]` entries moved to a new `[vX.Y.Z] - YYYY-MM-DD` section.
- [ ] `[Unreleased]` section reset to `(no entries yet)`.
- [ ] All Added/Changed/Fixed/Security sub-sections populated where relevant.
- [ ] `README.md` version badges and feature descriptions reflect the new version (if changed).
- [ ] No breaking changes without a corresponding major version bump.
- [ ] `npm audit --audit-level=high --omit=dev` exits 0.
- [ ] CI is green on the `main` branch before tagging.
- [ ] GitHub repository secrets `VSCE_PAT` and `OVSX_TOKEN` are valid and not expired.
- [ ] Tag format is `vX.Y.Z` (e.g., `v1.0.19`), annotated, not lightweight.
- [ ] VSIX file (if tested locally) is deleted and not staged.
- [ ] `docs/RELEASING.md` and `docs/CHECKLISTS.md` are up to date with any process changes.

---

## Post-Release Checklist

- [ ] GitHub Actions `Release` workflow completed with all steps green.
- [ ] VSIX attached to the GitHub Release page.
- [ ] Extension visible on VS Code Marketplace at the new version (allow 5-10 min).
- [ ] Extension visible on Open VSX at the new version (allow up to 15 min).
- [ ] Smoke-test: install the published version in a clean VS Code instance, add an annotation, reload, verify persistence.
- [ ] Monitor GitHub Issues for regressions over the next 24 hours.
